## 1. Overview

macOS 메뉴바에 상주하며 로컬에 있는 AI CLI(Claude Code, Codex, ...) **세션 히스토리를 한눈에** 보여주는 네이티브 앱. 세션 선택 시 해당 대화로 바로 resume, hover 시 최근 메시지 프리뷰.

### 1.1 Problem

- Claude Code / Codex 를 여러 프로젝트에서 사용하면 `/resume` 목록에서 "내가 마지막에 쓰던 그 대화" 찾기가 번거로움
- 각 CLI 마다 히스토리 위치가 다르고, 프로젝트별로 따로 저장되어 전체 조망 불가
- 어떤 세션이 지금 돌고 있는지(active), 최근에 썼는지(recent) 구분 어려움

### 1.2 Goals (v0.1 MVP)

| # | Goal | 측정 |
|---|---|---|
| G1 | Claude Code + Codex 세션을 **통합 리스트**로 메뉴바에 표시 | 로컬 JSONL 스캔, 200ms 이내 갱신 |
| G2 | 세션 메타정보 표시 | 프로젝트 경로, 첫 user 메시지(=타이틀), 마지막 수정 시각, CLI 종류 |
| G3 | **3가지 상태 구분** | 🟢 running (프로세스 살아있음) / 🟡 recent (최근 수정) / 🔘 stale |
| G4 | **Resume 원클릭** | 세션 행 클릭 → 기본 터미널에서 `claude --resume <id>` 또는 `codex resume <id>` 실행 |
| G5 | **Live Preview** | 행 hover 시 팝오버로 최근 N 메시지 (markdown + code highlight) |
| G6 | 파일시스템 감시 기반 **실시간 갱신** | 새 세션·메시지 추가 시 자동 반영 (폴링 X) |
| G7 | **CLI adapter 구조** | 새 CLI 추가 시 `SessionAdapter` 구현만으로 통합 |
| G8 | GitHub Releases + Homebrew tap 배포 | `brew install --cask 1989v/tap/aieye` |

### 1.3 Non-Goals (v0.1 out of scope)

- 세션 생성 (새 Claude Code 인스턴스 런칭) — 다른 터미널에서 하면 됨
- 토큰·사용량 통계 (기존 CodexBar/ClaudeBar 영역)
- Export (Markdown/HTML 변환) — v0.2
- Delete session — v0.2
- 세션 검색 / 필터 — v0.2 (MVP 는 단순 리스트)
- 원격 SSH 세션
- Mac App Store 배포 (샌드박스 제약)
- Windows 네이티브 (WSL 의 Claude Code/Codex 는 접근 경로 다름)

## 2. Architecture

### 2.1 Stack 선택 근거

- **Tauri v2** (Rust 백엔드 + Web 프론트엔드) — muxbar(Swift) 대비:
  - AI 채팅 렌더링(markdown/code/diff) HTML 이 절대 유리
  - macOS WKWebView 재사용 → 번들 가볍고 (~15MB) 메모리 효율
  - Linux 가능성 공짜로 따라옴
- **React + TypeScript** — 생태계 풍부, 채팅 UI 라이브러리 많음
- **Vite** — 빠른 HMR

### 2.4 Data Flow — Session Discovery

```
app start
  ↓
Rust SessionCoordinator::start()
  ↓
each adapter.initialScan() → Vec<Session>
  ↓
FS watch (notify crate) on adapter paths
  ↓
on new/modified JSONL → adapter.refresh(path) → Session
  ↓
Coordinator merge + dedupe → emit `sessions-changed` Tauri event
  ↓
Frontend useSessions() receives → re-render
```

### 2.5 Data Flow — Live Preview

```
React: user hovers SessionRow
  ↓
invoke("preview_session", {id, cli}) → Rust
  ↓
Adapter.parseLastMessages(path, N=20)
  ↓
returns Vec<Message> → serialized via serde
  ↓
Frontend renders with react-markdown + shiki (syntax highlight)
```

## 3. Core Components

### 3.1 SessionAdapter 트레이트 (Rust)

```rust
pub trait SessionAdapter: Send + Sync {
    fn cli(&self) -> CliKind;
    fn base_paths(&self) -> Vec<PathBuf>;

    async fn scan(&self) -> anyhow::Result<Vec<Session>>;
    async fn parse_session(&self, id: &str) -> anyhow::Result<SessionDetail>;
    async fn parse_last_messages(&self, id: &str, n: usize) -> anyhow::Result<Vec<Message>>;

    fn resume_command(&self, id: &str, cwd: Option<&Path>) -> Command;
}
```

CLI 별 구현: `ClaudeAdapter`, `CodexAdapter`.
신규 CLI 추가 시 트레이트 구현만으로 통합.

### 3.2 ClaudeAdapter

**Scan 전략**:
1. `~/.claude/projects/` 의 모든 slug 디렉터리 iterate
2. 각 `{sessionId}.jsonl` 파일에 대해:
   - `mtime` → lastActivity
   - JSONL 처음 ~10 라인 읽어 첫 `type:"user"` 메시지 추출 → title
   - 첫 라인의 `cwd` 추출 → project path
   - `gitBranch` 있으면 추출

**Project slug 디코딩**: `~/.claude/projects/-Users-gideok-kwon-IdeaProjects-msa` → `/Users/gideok-kwon/IdeaProjects/msa` (slash 복원).

**Resume 커맨드**:
```rust
// claude --resume <sessionId>
// cwd 로 cd 먼저
cd "<project path>" && claude --resume <sessionId>
```

### 3.3 CodexAdapter

**Scan 전략**:
1. `~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl` glob
2. 파일명에서 `sessionId` 추출 (uuid 부분)
3. 파일 처음 라인 파싱 → 첫 user prompt, cwd

**Resume 커맨드**: `codex resume <sessionId>` (Codex 버전별 지원 여부 확인 필요 → open question)

**Matching 전략** (running ↔ session):

- 정확히 매칭 불가능하면 "주어진 cwd 에서 CLI 실행 중" 수준으로 fallback

### 3.5 Session 모델

```rust
pub struct Session {
    pub id: String,              // sessionId (uuid 등)
    pub cli: CliKind,            // Claude | Codex | ...
    pub title: String,           // 첫 user 메시지 ~80자
    pub project_path: Option<PathBuf>,
    pub git_branch: Option<String>,
    pub jsonl_path: PathBuf,
    pub last_activity: DateTime<Utc>,
    pub message_count: Option<usize>, // 파일 라인 수 기반 (parsing cost ↓)
    pub state: SessionState,     // Running | Recent | Stale
}

pub enum SessionState {
    Running,                     // 프로세스 살아있음 + cwd 매칭
    Recent { within_minutes: u32 },  // mtime 기준
    Stale,
}
```

**Recent 임계값**: Settings 에서 조정 (기본 60분).

## 4. UX Details

### 4.1 Menu bar icon

- **Default**: SF Symbol-like `👁` (eye) 를 커스텀 PNG 로 제공 (template image)
- **Active sessions count badge**: 아이콘 옆 숫자 (muxbar 패턴)
- **Running indicator**: 초록 dot 오버레이 (프로세스 살아있는 세션 있을 때)

### 4.2 Click behavior

- **Row click**: Resume 실행 (즉시 터미널 열고 attach)
- **Row hover (300ms delay)**: Preview popover
- **⋯ menu**:
  - Reveal JSONL in Finder
  - Copy session ID
  - (v0.2) Export as Markdown
  - (v0.2) Delete

### 4.3 Resume 동작 상세

```
1. 선호 터미널 앱 확인 (Settings)
2. 프로젝트 경로로 cd + CLI resume 명령 조합
3. osascript 또는 `open -na TerminalApp --args ...` 로 실행
4. 실행된 터미널이 포커스되어 대화 이어짐
```

터미널별 adapter (muxbar 의 `TerminalAdapter` 와 유사):
- Terminal.app: AppleScript `do script`
- iTerm2: AppleScript / Python API
- Warp/Alacritty/kitty: `open -na` + `--args`

### 4.4 Empty / Loading / Error states

- 스캔 중: `Scanning your AI sessions…` + spinner
- 로컬에 세션 없음: `No sessions yet. Start a conversation in Claude Code or Codex.`
- 파일 접근 불가: `Cannot read ~/.claude — check permissions.`

## 5. Data Model

### 5.1 Rust types (serde serializable)

`src-tauri/src/sessions/model.rs` — `Session`, `SessionDetail`, `Message`, `SessionState` (3.5 참조).

### 5.2 TypeScript types

`src/types/session.ts` — Rust 와 동일한 shape. [ts-rs](https://github.com/Aleph-Alpha/ts-rs) 같은 툴로 자동 생성 (선택).

### 5.3 Tauri commands (IPC)

| command | args | returns |
|---|---|---|
| `list_sessions` | `{}` | `Session[]` |
| `preview_session` | `{id, cli}` | `Message[]` |
| `resume_session` | `{id, cli, terminal?}` | `()` |
| `reveal_in_finder` | `{path}` | `()` |
| `get_settings` | `{}` | `Settings` |
| `set_settings` | `{settings}` | `()` |

### 5.4 Tauri events (push)

| event | payload | 발행 시점 |
|---|---|---|
| `sessions-changed` | `Session[]` | FS watcher 가 변경 감지 |
| `running-changed` | `RunningCli[]` | ProcessDetector 주기 갱신 (5초) |

## 6. Error Handling & Edge Cases

| 케이스 | 처리 |
|---|---|
| JSONL 파싱 실패 | 해당 세션 skip + warning log, 나머지 계속 |
| `~/.claude` / `~/.codex` 디렉터리 없음 | 해당 adapter 비활성, UI 에 표시 |
| JSONL 파일이 쓰여지는 중 (partial write) | retry 후 여전히 실패면 skip |
| JSONL 파일 > 10MB | 처음/끝 N KB 만 읽어 메타데이터 추출 (full parse X) |
| 세션 ID 중복 (두 CLI 에 같은 uuid) | `{cli}-{id}` 조합 키로 구분 |
| Process cwd 조회 실패 (TCC 권한) | state=Recent 로 폴백 |
| 기본 터미널 미설치 | 에러 토스트 + Settings 유도 |
| Claude Code / Codex 바이너리 미설치 | Resume 버튼 비활성 + 툴팁 |
